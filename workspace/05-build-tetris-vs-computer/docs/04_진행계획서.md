# 테트리스 대결 진행 계획서

## 합의 원칙
하드코딩 우선 → 동작 확인 → 필요시 분리

---

## 마일스톤

### M1 — 지표 정의 (1일)
- 높이 합계 / 구멍 수 / 완성 줄 / 요철(bumpiness) 명세
- `evaluate(board) -> score` 인터페이스 확정
- **산출물:** 지표 정의서 + 함수 시그니처

### M2 — MVP 구현 (3일)
- 게임 로직 + AI 하드코딩
- UI: 보드 2개 나란히, 점수·레벨·다음 블록 표시
- **산출물:** 동작하는 대결 테트리스

---

## MVP vs v2 범위

| 항목 | MVP | v2 |
|------|-----|----|
| AI 가중치 | 하드코딩 | 설정 파일 분리 |
| AI vs AI | ❌ | 필요시 검토 |
| 튜닝 자동화 | ❌ | 필요시 검토 |
