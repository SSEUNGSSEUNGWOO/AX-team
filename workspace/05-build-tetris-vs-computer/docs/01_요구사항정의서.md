# 테트리스 대결 요구사항정의서

## 1. 프로젝트 개요
사용자(Human)와 휴리스틱 룰베이스 AI가 실시간으로 대결하는 테트리스 게임. AI 평가 함수는 모듈로 분리하고, 가중치는 설정 파일로 파라미터화한다.

---

## 2. 기능 요구사항

### 2-1. 게임 코어
| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F-01 | 표준 테트리스 규칙 구현 (7종 블록, 회전, 라인 클리어) | 🔴 필수 |
| F-02 | Human vs AI 동시 진행 (독립된 보드 2개) | 🔴 필수 |
| F-03 | 라인 클리어 시 상대방 보드에 방해 블록 전송 | 🟡 중요 |

### 2-2. AI 시스템
| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F-04 | AI 평가 지표 명세 확정 후 구현 시작 | 🔴 필수 |
| F-05 | 평가 지표: 집계높이(Aggregate Height), 구멍 수(Holes), 요철(Bumpiness), 완성 라인 수(Lines Cleared) | 🔴 필수 |
| F-06 | 평가 함수 인터페이스: `evaluate(board) -> score` 형태로 모듈 분리 | 🔴 필수 |
| F-07 | 가중치를 `ai_config.json` 설정 파일로 파라미터화 | 🟡 중요 |

> **⚠️ 제약 (F-04 관련):** 아래 지표 정의 합의 전 AI 코딩 시작 금지

#### AI 평가 지표 정의
| 지표명 | 정의 | 비고 |
|-------|------|------|
| Aggregate Height | 각 열 최고 블록 높이의 합계 | 낮을수록 좋음 |
| Holes | 블록 위쪽에 빈 셀이 존재하는 칸 수 | 낮을수록 좋음 |
| Bumpiness | 인접 열 높이 차이의 절댓값 합계 | 낮을수록 좋음 |
| Lines Cleared | 현재 수에 완성된 라인 수 | 높을수록 좋음 |

### 2-3. UI/UX
| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F-08 | Human 보드 / AI 보드 좌우 나란히 배치 | 🔴 필수 |
| F-09 | 각 보드별 점수·레벨·다음 블록 실시간 표시 | 🔴 필수 |
| F-10 | AI 현재 평가 점수 실시간 표시 (AI가 무엇을 보고 있는지 노출) | 🟡 중요 |

---

## 3. 비기능 요구사항
| ID | 요구사항 |
|----|---------|
| NF-01 | AI 수 계산은 블록 낙하 전 완료 (지연 없는 실시간 대결) |
| NF-02 | 게임 로직과 AI 모듈은 의존성 분리 (AI 교체 시 게임 로직 무수정) |
| NF-03 | 설정 파일 변경만으로 AI 가중치 튜닝 가능 |

---

## 4. 제약사항
| 항목 | 내용 |
|-----|------|
| 구현 순서 | 지표 정의 합의 → `evaluate()` 인터페이스 설계 → 하드코딩 구현 → 설정 파일 분리 |
| 스코프 아웃 | AI vs AI 시뮬레이션 프레임워크 (현재 불필요, 백로그 등록) |
| 스코프 아웃 | 머신러닝 기반 AI (현재 불필요) |

---

## 5. 우선순위 요약
```
🔴 필수 (MVP):
  F-01 게임 코어 → F-04/05 지표 정의 합의
  → F-06 evaluate() 모듈 분리 → F-02 AI 대결 → F-08/09 UI

🟡 중요 (v1.1):
  F-07 설정 파일 파라미터화
  F-03 방해 블록 전송
  F-10 AI 상태 실시간 표시

⚪ 백로그:
  AI vs AI 시뮬레이션 프레임워크 (주혁 제안, 추후 재논의)
```

> **지민 코멘트:** F-05 지표 정의는 위 표 기준으로 팀 합의 완료로 간주하겠습니다. 단, `Holes` 계산 시 **블록이 완전히 덮인 빈 셀만 카운트**하는지, **열 내 모든 빈 셀**인지 구현 전에 한 번 더 확인이 필요해요.
