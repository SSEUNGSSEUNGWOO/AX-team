## 3. AI 파이프라인

### 3-1. 지표 정의서 (지민 의견 반영 — 구현 전 합의 필수)

| 지표 | 정의 | 측정 방식 |
|------|------|-----------|
| `aggregateHeight` | 전체 열 높이 합산 | 각 열 최상단 블록 행 번호 합계 |
| `completeLines` | 완성된 줄 수 | 가로 10칸 모두 채워진 행 count |
| `holes` | 구멍 수 | 블록 아래 빈 셀 count |
| `bumpiness` | 인접 열 높이 차이 합 | `Σ|height[i] - height[i+1]|` |

> **합의 기준**: 위 4개 지표를 MVP 기준으로 확정. 추가 지표는 인터페이스 변경 없이 확장 가능.

### 3-2. AI 모듈 인터페이스 (수영 설계 — 핵심)

```typescript
// ai/types.ts
export interface BoardState {
  grid: number[][];        // 10x20 보드
  currentPiece: Piece;
  nextPiece: Piece;
}

export interface EvaluationResult {
  score: number;
  metrics: {
    aggregateHeight: number;
    completeLines: number;
    holes: number;
    bumpiness: number;
  };
}

export interface AIStrategy {
  evaluate(board: BoardState): EvaluationResult;
  selectMove(board: BoardState): Move;
}
```

```typescript
// ai/HeuristicAI.ts
import { AIStrategy, BoardState, EvaluationResult } from './types';
import { loadWeights } from './config';  // 설정 파일 연동

export class HeuristicAI implements AIStrategy {
  private weights = loadWeights();  // config/ai-weights.json

  evaluate(board: BoardState): EvaluationResult {
    const metrics = this.calcMetrics(board.grid);
    const score =
      this.weights.completeLines * metrics.completeLines -
      this.weights.aggregateHeight * metrics.aggregateHeight -
      this.weights.holes * metrics.holes -
      this.weights.bumpiness * metrics.bumpiness;

    return { score, metrics };
  }

  selectMove(board: BoardState): Move {
    // 가능한 모든 배치 시뮬레이션 → 최고 점수 선택
    return this.getBestPlacement(board);
  }

  private calcMetrics(grid: number[][]) { /* ... */ }
  private getBestPlacement(board: BoardState): Move { /* ... */ }
}
```

```json
// config/ai-weights.json  ← 설정 파일 파라미터화 (팀 합의 반영)
{
  "completeLines": 0.76,
  "aggregateHeight": -0.51,
  "holes": -0.36,
  "bumpiness": -0.18
}
```

> **승우 의견 반영**: 초기엔 위 JSON이 하드코딩 수준으로 단순하게 시작.  
> **수영 설계 의도**: 단, `evaluate()` 인터페이스는 지금 잡아둬야 게임 엔진 재수정 없이 알고리즘 교체 가능.

### 3-3. AI vs AI 시뮬레이션 프레임워크 (주혁 의견 — 구조만 예약)

```typescript
// simulation/AISimulator.ts  ← 지금 당장 구현 X, 인터페이스만 예약
export interface Simulator {
  run(strategyA: AIStrategy, strategyB: AIStrategy, rounds: number): SimResult;

```

> **주혁 의견 반영**: 단, 인터페이스 예약으로 나중에 붙이기 용이하게 구조 확보.
