# 할일 관리 웹앱 요구사항 정의서

## 1. 프로젝트 개요
바닐라 JS 기반 할일 관리 웹앱. 추가/완료/삭제 기능과 로컬스토리지 영속성 제공.

---

## 2. 기능 요구사항

### 2-1. 데이터 모델
```js
{
  id: string,        // crypto.randomUUID()
  title: string,
  status: 'active' | 'done',
  createdAt: string  // ISO8601
}
```
> ⚠️ `version` 필드: **미포함** — YAGNI 원칙 적용 (유진 의견 채택, 주혁 의견 보류)

### 2-2. 핵심 기능
| 기능 | 설명 |
|------|------|
| 할일 추가 | 입력값 trim 후 빈 문자열이면 추가 불가 |
| 완료 처리 | status 토글, 시각적 구분 표시 |
| 삭제 | 항목 제거 |
| 영속성 | 로컬스토리지 read/write 유틸 함수로 처리 |

### 2-3. UI/UX (민아)
- 입력창 비었을 때 추가 버튼 **비활성화**
- 완료 항목: 취소선 + 회색 처리로 시각 구분

---

## 3. 비기능 요구사항

### 3-1. 검증 로직 (지민 + 수영 — 필수)
```js
// 별도 함수로 분리 — 타협 없음
function validateTodo(data) {
  return {
    id: data?.id ?? crypto.randomUUID(),
    title: typeof data?.title === 'string' ? data.title.trim() : '',
    status: ['active','done'].includes(data?.status) ? data.status : 'active',
    createdAt: data?.createdAt ?? new Date().toISOString()
  };
}
```
> 로컬스토리지 **read 시 반드시 통과** → 오염 데이터 차단

### 3-2. 로컬스토리지 유틸 (승우)
```js
function readStorage() { ... }   // parse + validateTodo 적용
function writeStorage(todos) { ... }  // stringify
```

---

## 4. 제약사항
- 바닐라 JS 전용 (프레임워크/Zod 금지)
- null 체크 + 기본값 fallback은 순수 JS로
- 검증 로직은 **독립 함수 분리 필수** (인라인 작성 금지)

---

## 5. 우선순위

| 순위 | 항목 | 근거 |
|------|------|------|
| P0 | 데이터 모델 + `validateTodo()` 정의 | 이후 모든 로직의 기반 |
| P0 | 로컬스토리지 read/write 유틸 | 영속성 핵심 |
| P1 | 추가/완료/삭제 기능 | 핵심 사용자 기능 |
| P1 | 입력 버튼 비활성화 | UX 기본 |
| P2 | 완료 항목 시각 구분 | 가독성 |
| 보류 | `version` 필드 | 현시점 불필요, 스키마 변경 시 재검토 |

---

> **검증 미적용 상태로 다음 단계 진행 금지.** `validateTodo()` 구현 완료 후 스토리지 연동 진행.
