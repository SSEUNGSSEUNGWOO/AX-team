# 할일 관리 웹앱 기술 설계서

## 1. 데이터 모델

```js
// models/todo.js
{
  id: string,        // crypto.randomUUID()
  title: string,     // null 불가
  status: 'active' | 'done',
  createdAt: string  // ISO 8601
  // version: YAGNI — 미포함 (유진 의견 반영)
}
```

> `version` 필드는 현재 미포함. 스키마 변경 시점에 마이그레이션 유틸 추가 대응.

## 4. 핵심 모듈 설계

### 4-1. validateTodo()

```js
// validators/validateTodo.js
export function validateTodo(data) {
  if (!data || typeof data !== 'object') return null;

  return {
    id:        typeof data.id === 'string'    ? data.id        : crypto.randomUUID(),
    title:     typeof data.title === 'string' && data.title.trim()
                 ? data.title.trim()          : null,           // null → 필터링
    status:    ['active', 'done'].includes(data.status)
                 ? data.status                : 'active',
    createdAt: data.createdAt ?? new Date().toISOString()
  };
}
// title === null인 항목은 저장·렌더 단계에서 즉시 제거
// → 로컬스토리지 오염 데이터가 앱 전체를 깨뜨리는 상황 원천 차단
```

### 4-2. storageUtils.js

```js
// storage/storageUtils.js
const KEY = 'todos_v1';

export function readTodos() {
  try {
    const raw = JSON.parse(localStorage.getItem(KEY) ?? '[]');
    return Array.isArray(raw)
      ? raw.map(validateTodo).filter(t => t?.title !== null)
      : [];
  } catch {
    return [];  // 파싱 실패 시 빈 배열 fallback
  }
}

export function writeTodos(todos) {
  localStorage.setItem(KEY, JSON.stringify(todos));
}
```

> read/write 2개 유틸로 단순하게 — 단, read 시 **반드시 validateTodo 통과** 후 저장.

### 4-3. UI / renderer.js

```js
// ui/renderer.js
export function renderTodos(todos) {
  list.innerHTML = '';
  todos.forEach(todo => {
    const li = document.createElement('li');
    li.className = todo.status === 'done' ? 'done' : '';
    // 완료 항목: 취소선 + 회색 처리 (민아 의견)
    li.innerHTML = `
      <span>${todo.title}</span>
      <button data-id="${todo.id}" data-action="toggle">완료</button>
      <button data-id="${todo.id}" data-action="delete">삭제</button>
    `;
    list.appendChild(li);
  });
}

// 입력창 비었을 때 추가 버튼 비활성화 (민아 의견)
input.addEventListener('input', () => {
  addBtn.disabled = input.value.trim() === '';
});
```

## 5. 이벤트 흐름

```
사용자 입력
    ↓
[app.js] 이벤트 수신
    ↓
validateTodo() → title null이면 즉시 차단
    ↓
writeTodos() → localStorage 저장
    ↓
renderTodos() → DOM 갱신
```

## 7. 의견 반영 요약

| 팀원 | 의견 | 반영 여부 |
|------|------|-----------|
| 승우 | read/write 유틸 2개 | ✅ storageUtils.js |
| 지민 | validateTodo() 처음부터 분리 | ✅ validators/ 모듈 |
| 주혁 | version 필드 추가 | ❌ YAGNI (유진 의견 채택) |
| 유진 | version 미포함, 필요 시 대응 | ✅ 현재 스키마 제외 |
| 수영 | validateTodo 필수, 오염 차단 | ✅ read 시 강제 통과 |
| 민아 | 버튼 비활성화 + 시각 구분 | ✅ renderer.js + CSS |

## 8. 확장 고려사항

- 스키마 변경 시 → `storageUtils.js`에 마이그레이션 함수 추가, KEY를 `todos_v2`로 교체
- 서버 연동 시 → `storageUtils.js`를 API 클라이언트로 교체만 하면 나머지 모듈 무변경
- 필터링(전체/진행/완료) 추가 시 → `renderer.js`에 필터 파라미터
