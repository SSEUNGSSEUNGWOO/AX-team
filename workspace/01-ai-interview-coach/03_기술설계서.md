## 3. API 설계

### 3.1 Base URL
```
https://api.interviewcoach.io/v1
```

### 3.2 인증

```
POST   /auth/register          # 회원가입
POST   /auth/login             # 로그인 → JWT 발급
POST   /auth/refresh           # 토큰 갱신
DELETE /auth/logout            # 로그아웃
```

### 3.3 질문 생성

```
POST   /questions/generate
```

**Request Body:**
```json
{
  "job_category": "backend_engineer",
  "job_level": "mid",
  "tech_stack": ["Python", "FastAPI", "PostgreSQL"],
  "question_count": 5,
  "question_type": ["technical", "behavioral"]
}
```

**Response:**
```json
{
  "session_id": "uuid-v4",
  "questions": [
    {
      "id": "q_001",
      "type": "technical",
      "category": "database",
      "content": "PostgreSQL에서 인덱스 설계 시 고려사항을 설명해주세요.",
      "difficulty": "medium",
      "follow_up_hints": ["B-tree vs Hash", "복합 인덱스"]
    }
  ],
  "generated_at": "2025-01-15T09:00:00Z"
}
```

### 3.4 답변 피드백

```
POST   /feedback/analyze
```

**Request Body:**
```json
{
  "session_id": "uuid-v4",
  "question_id": "q_001",
  "answer": {
    "text": "사용자 답변 텍스트",
    "audio_url": "s3://bucket/answer.mp3",
    "duration_seconds": 120
  }
}
```

**Response:**
```json
{
  "feedback_id": "fb_001",
  "scores": {
    "accuracy": 78,
    "completeness": 65,
    "clarity": 82,
    "overall": 75
  },
  "strengths": ["핵심 개념 정확히 파악", "구체적 예시 사용"],
  "improvements": ["B-tree 인덱스 내부 구조 설명 보완 필요"],
  "model_answer_summary": "이상적 답변 요약...",
  "follow_up_question": "그렇다면 인덱스 스캔 vs 풀 스캔 결정 기준은?"
}
```

### 3.5 세션 관리

```
GET    /sessions                # 내 세션 목록
GET    /sessions/{session_id}   # 세션 상세 (질문+피드백 전체)
DELETE /sessions/{session_id}   # 세션 삭제
GET    /sessions/{session_id}/report  # 최종 리포트 생성
```

### 3.6 직군 카탈로그

```
GET    /categories              # 직군 목록 전체
GET    /categories/{id}/skills  # 직군별 기술 스택 목록
```

## 4. DB 구조

```sql
-- 사용자
CREATE TABLE users (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email       VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    name        VARCHAR(100),
    created_at  TIMESTAMPTZ DEFAULT NOW(),
    updated_at  TIMESTAMPTZ DEFAULT NOW()
);

-- 직군 카탈로그
CREATE TABLE job_categories (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    slug        VARCHAR(100) UNIQUE NOT NULL,  -- 'backend_engineer'
    name        VARCHAR(100) NOT NULL,
    description TEXT,
    is_active   BOOLEAN DEFAULT TRUE
);

-- 면접 세션
CREATE TABLE interview_sessions (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID REFERENCES users(id) ON DELETE CASCADE,
    job_category_id UUID REFERENCES job_categories(id),
    job_level       VARCHAR(50),   -- junior / mid / senior
    tech_stack      JSONB,         -- ["Python", "FastAPI"]
    status          VARCHAR(50) DEFAULT 'in_progress',
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    completed_at    TIMESTAMPTZ
);

-- 질문
CREATE TABLE questions
