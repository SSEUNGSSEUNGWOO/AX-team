# 쿠팡 가격 모니터링 툴 — 시장조사 보고서

---

## 1. 시장 규모

| 구분 | 규모 | 근거 |
|------|------|------|
| **TAM** | ~2.3조 원 | 국내 이커머스 가격비교·알림 서비스 전체 시장 |
| **SAM** | ~1,200억 원 | 쿠팡 활성 이용자 기반 가격추적 수요층 (쿠팡 MAU 약 3,000만 기준) |
| **SOM** | ~12억 원 | 초기 3년 내 침투 가능 추정치 (SAM의 1%) |

> ⚠️ TAM 수치는 오픈서베이·통계청 이커머스 보고서 기반 추정. 별도 검증 권장.

---

## 2. 경쟁사 분석

| 서비스 | 특징 | 약점 |
|--------|------|------|
| **에누리닷컴** | 멀티플랫폼 가격비교, 히스토리 제공 | 쿠팡 실시간성 약함 |
| **다나와** | 하드웨어 중심 비교 강점 | 카테고리 편중 |
| **Honey (미국)** | 브라우저 익스텐션, 쿠폰 자동적용 | 국내 쿠팡 미지원 |
| **크롤링 오픈소스** | GitHub 다수 존재 (비공개 운영) | 유지보수 없음, 차단 리스크 |

**결론:** 쿠팡 특화 + 목표가 알림 조합은 틈새 존재. 단, 쿠팡 자체 `알림받기` 기능과 직접 경쟁.

---

## 3. 타겟 고객

```
Primary  → 쿠팡 헤비유저 (월 3회 이상 구매, 가격 민감형)
Secondary → 셀러/리셀러 (경쟁상품 가격 추적 목적)
Tertiary  → 기업 구매팀 (소모품 대량구매 타이밍 포착)
```

---

## 4. 수익 모델

| 모델 | 단가 | 마진 구조 |
|------|------|-----------|
| **SaaS 구독** | 월 3,900~9,900원 | 서버비 제외 마진 70%↑ |
| **쿠팡 파트너스 연동** | 구매 전환 시 CPS 1~3% | 추가 개발 없이 수익화 가능 |
| **B2B API 제공** | 월 50~200만 원/사 | 셀러 툴 연동 시 고단가 |

> 파트너스 CPS 모델이 초기 수익화에 가장 현실적.

---

## 5. 리스크

```
🔴 HIGH
  - 쿠팡 크롤링 차단 (UA 감지, IP 밴, DOM 구조 변경)
    → robots.txt 위반 가능성, 법적 리스크 동반

🟠 MEDIUM  
  - 쿠팡 자체 알림 기능 고도화 시 차별점 소멸
  - 가격 데이터 정합성 오류 → 사용자 신뢰 하락

🟡 LOW
  - 서버 비용 급증 (모니터링 주기·상품 수 비례)
  - 파트너스 정책 변경으로 CPS 수익 축소
```

---

## 6. 종합 판단

> **수요 있음 / 수익화 가능 / 단, 기술 리스크가 핵심 변수**
>
> 쿠팡 공식 API 미제공 상태에서 크롤링 의존도가 높아 **안정성·법적 검토가 선행**되어야 함.
> 파트너스 CPS 연동 + 구독 혼합 모델로 초기 수익화 권장.
