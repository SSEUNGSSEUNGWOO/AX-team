## 2. 기술 스택

| 레이어 | 기술 | 선택 이유 |
|--------|------|-----------|
| 런타임 | Python 3.11+ | 생태계, 스크래핑 친화적 |
| CLI | `argparse` + `rich` | 표준 + 콘솔 UI 품질 |
| HTTP | `requests` + `fake-useragent` | 봇 탐지 우회 기본 |
| 파싱 | `BeautifulSoup4` + `lxml` | 안정적 HTML 파싱 |
| DB | `SQLite` + `SQLAlchemy 2.0` | 로컬 툴, ORM 유지보수성 |
| 스케줄러 | `APScheduler` | 경량 인프라 내 주기 실행 |
| 설정 | `python-dotenv` + `pydantic-settings` | 환경별 설정 분리 |
| 테스트 | `pytest` + `responses` | HTTP mock 포함 |

> **Anti-pick:** Selenium/Playwright → 현 단계 오버킬. 봇 탐지 강화 시 2단계에서 도입 검토.

## 4. DB 스키마

### 4-1. `products` 테이블

```sql
CREATE TABLE products (
    id          INTEGER     PRIMARY KEY AUTOINCREMENT,
    url         TEXT        NOT NULL UNIQUE,
    name        TEXT,                          -- 최초 스크래핑 시 자동 저장
    target_price INTEGER    NOT NULL,          -- 원 단위 정수
    is_active   BOOLEAN     NOT NULL DEFAULT 1,
    created_at  DATETIME    NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME    NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

| 컬럼 | 타입 | 제약 | 설명 |
|------|------|------|------|
| `url` | TEXT | UNIQUE NOT NULL | 쿠팡 상품 고유 URL |
| `target_price` | INTEGER | NOT NULL | 목표가 (원, 소수점 없음) |
| `is_active` | BOOLEAN | DEFAULT 1 | soft delete / 일시 중단 |
| `name` | TEXT | nullable | 첫 스크래핑 후 자동 채움 |

---

### 4-2. `price_logs` 테이블

```sql
CREATE TABLE price_logs (
    id          INTEGER  PRIMARY KEY AUTOINCREMENT,
    product_id  INTEGER  NOT NULL REFERENCES products(id) ON DELETE CASCADE,
    price       INTEGER  NOT NULL,             -- 스크래핑 시점 가격
    is_alerted  BOOLEAN  NOT NULL DEFAULT 0,   -- 알림 발송 여부
    scraped_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_price_logs_product_id ON price_logs(product_id);
CREATE INDEX idx_price_logs_scraped_at ON price_logs(scraped_at);
```

| 컬럼 | 설명 |
|------|------|
| `price` | 해당 시점 실제 가격 |
| `is_alerted` | 목표가 달성 알림 중복 방지 플래그 |
