# 쿠팡 가격 모니터링 툴 — 진행 계획서

---

## 📌 개요

| 항목 | 내용 |
|------|------|
| 목적 | 쿠팡 상품 가격 자동 모니터링 및 목표가 도달 시 콘솔 알림 |
| 언어 | Python 3.11+ |
| 저장소 | SQLite (로컬), 추후 PostgreSQL 마이그레이션 고려 |
| 실행 방식 | CLI + 스케줄러 (APScheduler) |

---

## 🗃️ DB 스키마

```sql
-- 상품 등록 테이블
CREATE TABLE products (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    url         TEXT    NOT NULL UNIQUE,
    name        TEXT,
    target_price REAL   NOT NULL,
    is_active   INTEGER NOT NULL DEFAULT 1,  -- 0: 비활성
    created_at  TEXT    NOT NULL DEFAULT (datetime('now')),
    updated_at  TEXT    NOT NULL DEFAULT (datetime('now'))
);

-- 가격 이력 테이블
CREATE TABLE price_history (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    product_id  INTEGER NOT NULL REFERENCES products(id),
    price       REAL    NOT NULL,
    crawled_at  TEXT    NOT NULL DEFAULT (datetime('now'))
);

-- 알림 이력 테이블
CREATE TABLE alert_log (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    product_id  INTEGER NOT NULL REFERENCES products(id),
    triggered_price REAL NOT NULL,
    target_price    REAL NOT NULL,
    alerted_at  TEXT    NOT NULL DEFAULT (datetime('now'))
);
```

## 🔧 CLI API 엔드포인트 (명령어 인터페이스)

```
python main.py add    --url <URL> --target <가격>   # 상품 등록
python main.py list                                 # 등록 상품 목록 조회
python main.py remove --id <ID>                     # 상품 삭제
python main.py update --id <ID> --target <새목표가> # 목표가 수정
python main.py run    [--interval 60]               # 모니터링 시작 (초단위)
python main.py history --id <ID>                    # 가격 이력 조회
```

## 🏗️ 아키텍처

```
main.py (CLI Entry)
│
├── cli/
│   └── commands.py         # argparse 기반 명령어 파싱
│
├── core/
│   ├── crawler.py          # requests + BeautifulSoup 크롤링
│   ├── scheduler.py        # APScheduler 래퍼
│   └── notifier.py         # 콘솔 알림 출력 (추후 확장 포인트)
│
├── db/
│   ├── connection.py       # SQLite 연결 관리
│   ├── models.py           # 테이블 생성/마이그레이션
│   └── repository.py       # CRUD 쿼리 분리
│
└── config.py               # 환경변수, 상수 관리
```

> **설계 원칙**: crawler ↔ scheduler ↔ notifier 각 레이어 단방향 의존. 나중에 notifier만 교체해도 나머지 안 건드림.

## 🚀 마일스톤

### ✅ Milestone 1 — 기반 구축 (Day 1~2)

| 항목 | 완료 기준 |
|------|-----------|
| DB 초기화 (`models.py`) | `products`, `price_history`, `alert_log` 테이블 자동 생성 확인 |
| Repository CRUD 구현 | 단위 테스트 통과 (add/list/remove/update) |
| 환경 설정 (`config.py`) | `.env` 로드, 크롤 주기 기본값 설정 완료 |

---

### ✅ Milestone 2 — 크롤러 구현 (Day 3~4)

| 항목 | 완료 기준 |
|------|-----------|
| `crawler.py` 구현 | 쿠팡 상품 URL → 가격 파싱 성공 (수동 테스트 3건 이상) |
| User-Agent 로테이션 | 차단 방지 헤더 설정 확인 |
| 파싱 실패 예외 처리 | 실패 시 `price_history` 미기록, 오류 로그 출력 확인 |

> ⚠️ **리스크**: 쿠팡 DOM 변경 시 파서 깨짐. CSS 셀렉터를 `config.py`에 외부화해서 코드 수정 없이 대응 가능하게 설계.

---

### ✅ Milestone 3 — CLI + 스케줄러 (Day 5~6)

| 항목 | 완료 기준 |
|------|-----------|
| `commands.py` 전체 명령어 구현 | `add/list/remove/update/run/history` 6개 명령 정상 동작 |
| `scheduler.py` 주기 실행 | `--interval` 옵션으로 주기 변경 가능, 실행 중 Ctrl+C 정상 종료 |
| `notifier.py` 알림 출력 | 목표가 이하 시 콘솔 포맷 출력 + `alert_log` 기록 확인 |

  시각   : 2025-07-10 14:32:01
