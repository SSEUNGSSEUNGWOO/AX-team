## 2. 기술 스택

| 레이어 | 기술 | 선택 이유 |
|--------|------|-----------|
| **Frontend** | Next.js 14 (App Router) | SSR·SEO, 지도 렌더링 최적화 |
| **Backend** | FastAPI (Python 3.11) | AI 파이프라인 동일 언어, 비동기 처리 |
| **AI/ML** | scikit-learn · XGBoost · pandas | MVP 경량 모델 우선 (RAG는 v2) |
| **DB (관계형)** | PostgreSQL 15 | 상권 다차원 쿼리, PostGIS 공간 인덱스 |
| **캐시** | Redis 7 | 자주 조회되는 상권 데이터 TTL 캐싱 |
| **파일 스토리지** | AWS S3 | PDF 리포트 저장 |
| **작업 큐** | Celery + Redis | 비동기 파이프라인 실행 |
| **인프라** | Docker Compose → ECS (Fargate) | 로컬 개발·운영 일관성 |
| **지도** | Kakao Maps API | 국내 상권 데이터 정합성 |

## 3. AI 파이프라인 (MVP 3단계)

> ⚠️ **팀 합의 원칙**: 데이터 정제 기준 문서 완료 후 코딩 시작. RAG는 v2.

```
[Step 1 — 수집 (Collector)]
  ├─ 외부 API: 소상공인시장진흥공단 상권정보 API
  ├─ 외부 API: 행정안전부 인구통계 API
  ├─ 내부 DB : 가맹본부 제공 출점 이력 CSV 업로드
  └─ 스케줄: Celery Beat 일 1회 자동 수집
        │
        ▼ raw_data 테이블 저장
        
[Step 2 — 정제 (Refiner)]
  ├─ 결측치 처리
  │    ├─ 수치형: 컬럼별 결측률 > 30% → 해당 레코드 제외 (문서화 기준 #R-01)
  │    └─ 범주형: 최빈값 대체 (결측률 ≤ 10%), 초과 시 제외 (#R-02)
  ├─ 이상치 처리
  │    ├─ IQR 방식: Q1-1.5×IQR ~ Q3+1.5×IQR 범위 외 → 클리핑 (#R-03)
  │    └─ 유동인구 음수값 → 즉시 제외 (#R-04)
  ├─ 정규화: Min-Max (지역 단위 기준)
  └─ 결과: clean_data 테이블 저장 + 정제 로그 audit_log 테이블 기록
        │
        ▼
        
[Step 3 — 출력 (Reporter)]
  ├─ 모델: XGBoost 회귀 (출점 성공 점수 0~100)
  ├─ 피처 (v1): 유동인구·경쟁점 수·임대료 지수·매출 추정·업종 적합도
  ├─ 출력: JSON 스코어 + 근거 피처 Top 5 (SHAP values)
  └─ 리포트: PDF 자동 생성 → S3 업로드 → presigned URL 반환
```

### 데이터 정제 기준표 (문서화 우선)

| 규칙 ID | 대상 | 조건 | 처리 방법 |
|---------|------|------|-----------|
| R-01 | 수치형 전체 | 컬럼 결측률 > 30% | 레코드 제외 |
| R-02 | 범주형 전체 | 결측률 ≤ 10% | 최빈값 대체 |
| R-02 | 범주형 전체 | 결측률 > 10% | 레코드 제외 |
| R-03 | 수치형 전체 | IQR 범위 외 | 클리핑 (상·하한 대체) |
| R-04 | 유동인구 | 음수 | 즉시 제외 + 경고 로그 |
| R-05 | 임대료 | 0원 | 결측 처리 후 R-01 적용 |

## 4. DB 스키마

```sql
-- 원천 데이터
CREATE TABLE raw_data (
    id            BIGSERIAL PRIMARY KEY,
    source        VARCHAR(50)  NOT NULL,          -- 'sgis' | 'mois' | 'upload'
    collected_at  TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    payload       JSONB        NOT NULL,           -- 원본 그대로 보존
    status        VARCHAR(20)  NOT NULL DEFAULT 'pending'
                  CHECK (status IN ('pending','processed','failed'))
);

-- 정제 데이터
CREATE TABLE clean_data (
    id               BIGSERIAL PRIMARY KEY,
    raw_id           BIGINT REFERENCES raw_data(id),
    region_code      VARCHAR(10)  NOT NULL,        -- 행정동 코드
    geom             GEOGRAPHY(Point, 4326),       -- PostGIS 좌표
