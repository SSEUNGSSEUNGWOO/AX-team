# 요구사항정의서 — SpotMind

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | SpotMind — AI 기반 B2B 프랜차이즈 입지 분석 플랫폼 |
| 목적 | 과거 출점 이력·심사 기준·상권 데이터를 AI로 종합, 출점 권고 리포트 자동 생성 |
| 대상 고객 | 프랜차이즈 본사 출점 담당자, 가맹 희망자 |
| MVP 범위 | **파이프라인 3단계(수집→정제→출력)** 에 한정. RAG·고도화 기능 제외 |

---

## 2. 기능 요구사항

### 2.1 MVP — 파이프라인 3단계

#### STEP 1. 데이터 수집 (Collect)

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| F-01 | 공공 API(소상공인 상권정보, 행정안전부 인구통계)를 통해 상권 데이터 자동 수집 | **P0** |
| F-02 | 프랜차이즈 본사가 과거 출점 이력(위치·매출·폐점여부) CSV/엑셀 업로드 가능 | **P0** |
| F-03 | 수집 작업 스케줄링(일 1회 배치) 및 수집 로그 저장 | **P0** |
| F-04 | 수집 실패 시 알림(이메일 또는 슬랙 웹훅) 발송 | **P1** |

> ⚠️ **정의 필요:** F-02 파일 허용 사이즈 상한, 컬럼 스키마 표준안 문서화 먼저 완료 후 구현 시작.

---

#### STEP 2. 데이터 정제 (Refine)

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| F-05 | **결측치 처리:** 필드별 결측률 임계값(기본값 30%) 초과 시 해당 레코드 제외 처리 | **P0** |
| F-06 | **이상치 탐지:** IQR 기반으로 수치 필드 이상치 플래그 지정 및 로그 기록 | **P0** |
| F-07 | 주소 → 위경도 좌표 변환(지오코딩) 및 행정동 코드 매핑 | **P0** |
| F-08 | 중복 레코드 탐지(동일 위치·동일 브랜드 기준) 및 자동 제거 | **P0** |
| F-09 | 정제 결과 리포트(처리 건수, 제외 건수, 이상치 건수) 저장 | **P1** |

> ⚠️ **팀 합의 이행:** 결측치 임계값·이상치 규칙을 **`data_quality_spec.md`** 로 문서화 완료 후 F-05~F-08 코딩 착수.

---

#### STEP 3. 출력 (Output)

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| F-10 | 입력 주소(또는 좌표) 기준 반경 설정(500m / 1km / 2km) 상권 분석 실행 | **P0** |
| F-11 | 출점 권고 점수(0~100) 및 근거 항목(유동인구·경쟁밀도·유사 성공 사례 수) 계산 | **P0** |
| F-12 | 출점 권고 리포트 PDF 자동 생성 (브랜드 로고·지도 포함) | **P0** |
| F-13 | 리포트 히스토리 저장 및 재다운로드 기능 | **P1** |
| F-14 | 분석 결과 지도 시각화 (반경 내 경쟁점·유동인구 히트맵 오버레이) | **P1** |

---

### 2.2 Post-MVP (RAG 단계 — 이번 문서 범위 외)

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| F-20 | 자연어 질의 기반 입지 조건 검색 (RAG + LLM) | **P2** |
| F-21 | 심사 기준 문서 업로드 → 벡터 DB 색인 | **P2** |

## 3. 비기능 요구사항

| ID | 항목 | 요구사항 | 기준 |
|----|------|----------|------|
| NF-01 | 성능 | 단일 분석 요청 응답 시간 | **≤ 10초** (P95) |
| NF-02 | 가용성 | 서비스 가동률 | **99.5% 이상** (월 기준) |
| NF-03 | 보안 | 고객사 데이터(출점 이력) 격리 | 테넌트 단위 데이터 접근 제어 필수 |
| NF-04 | 보안 | 전송 구간 암호화 | TLS 1.2 이상 |
| NF-05 | 확장성 | 동시 분석 요청 처리 | **50 req/min** 이상 (MVP 기준) |
| NF-06 | 유지보수 | 정제 규칙 변경 시 재배포 없이 적용 | 설정 파일(YAML) 기반 규칙 관리 |
| NF-07 | 감사 | 모든 데이터 수집·정제·출력 이벤트 | 로그 보존 **90일** 이상 |
| NF-08 | 접근성 | 웹 UI | Chrome 최신 버전 기준, 모바일 반응형 |
