# arXiv 논문 수집 에이전트 기술 설계서

## 팀 합의 요약

| 쟁점 | 합의 결과 |
|------|-----------|
| Claude API 실패 처리 | 재시도 1회(3초 대기) + 원본 초록 폴백 저장 |
| config 파싱 실패 | `sys.exit(1)` — 조용한 폴백 없음 |
| 중복 방지 | 파일 스캔 방식 (DB 없음) |
| 에러 정책 설정 | `config.yaml`에 `error_handling` 블록으로 명시화 |
| 실패 논문 가시성 | MD 내 `## ⚠️ 요약 실패 논문` 섹션 + `ERROR_YYYY-MM-DD.md` |
| 재수집 메커니즘 | `output/retry_queue.json`으로 실패 논문 추적 |
| 모니터링 | `output/stats/YYYY-MM-DD.json` 수집 통계 저장 |
| 파일 구조 | 3개 핵심 파일 원칙 (agent.py / summarizer.py / config.yaml) |

## 3. 기술 스택

| 구성요소 | 선택 | 선택 이유 |
|----------|------|-----------|
| arXiv 수집 | `feedparser` | 공식 요구사항 |
| AI 요약 | `anthropic` (Claude API) | 공식 요구사항 |
| 설정 관리 | `PyYAML` | 경량, 가독성 |
| 스케줄링 | OS cron / GitHub Actions | 에이전트 자체는 스케줄러 불포함. 외부 트리거로 실행 |
| 로깅 | Python 표준 `logging` | 외부 의존성 추가 없음 |
| 상태 저장 | JSON 파일 | DB 없음 — 현재 규모에서 과설계 |
| 중복 방지 | 파일 스캔 (arXiv ID 파싱) | DB 없이 신뢰성 확보 |

> **수영 주석**: DB 도입은 "하루 수백 건 이상 수집" 요구사항이 생겼을 때 검토. 지금은 파일 스캔으로 충분하고, 교체 비용도 낮음.

## 4. config.yaml 전체 스펙

```yaml
# arXiv 수집 설정
arxiv:
  keywords:
    - LLM
    - multi-agent
    - RAG
  max_papers: 10                 # 하루 최대 수집 논문 수 (기본값 10)
  search_category: "cs.AI"       # arXiv 카테고리 필터 (선택)

# Claude API 설정
claude:
  model: "claude-3-5-sonnet-20241022"
  max_tokens: 500
  summary_language: "ko"

# 에러 처리 정책 (수영 요구사항: 정책을 코드가 아닌 설정으로 명시)
error_handling:
  claude_api:
    strategy: "retry_once"       # retry_once | skip_with_original | fail_fast
    retry_delay_seconds: 3
  config_parse_failure: "exit"   # 고정값. 변경 불가 (코드 레벨 강제)

# 출력 설정
output:
  format: "md"
  directory: "output"
  stats_enabled: true            # output/stats/ 통계 저장 여부
  retry_queue_enabled: true      # 실패 논문 재시도 큐 활성화
```

## 5. 에러 처리 정책 (팀 합의 핵심)

### 5-1. Config 파싱 실패

```
YAML 문법 오류 감지
       │
       ▼
logger.error("config.yaml 파싱 실패: {e}")
       │
       ▼
output/ERROR_{날짜}.md 생성
(에러 내용 + 수정 가이드 포함)
       │
       ▼
sys.exit(1)  ← 조용한 폴백 없음 (팀 전원 합의)
```

### 5-2. Claude API 실패 처리 플로우

```
Claude API 호출 (1차 시도)
       │
  실패 ├─────────────────────────────────────┐
       │                                     │
  성공 │                              logger.warning(
       │                              f"[RETRY] {title}: {e}"
       │                              )
       │                                     │
       ▼                              3초 대기
  요약 반환                                  │
                                    Claude API 호출 (2차 시도)
                                             │
                                   실패 ─────┴───── 성공
                                    │               │
                            logger.error(      요약 반환
                            f"[FAILED] {title}: {e}"
                            )
                                    │
                            원본 초록 반환
                            + retry_queue.json에 추가
```

### 5-3. MD 파일 내 실패 논문 표현

```markdown
# 📚 arXiv 논문 수집 결과 — 2025-01-15

**수집 통계**: 전체 10개 | ✅ 성공 8개 | ⚠️ 요약 실패 2개

---

## ✅ 요약 성공 논문

### 1. [논문 제목]
...

---

## ⚠️ 요약 실패 논문 (원본 초록 저장)

> Claude API 요약 실패. 아래는 원본 초록입니다. retry_queue.json에 추가되었습니다.

### 1. [논문 제목]
- **저자**: ...
- **링크**: ...
- **원본 초록**: ...
```

## 7. AI 파이프라인 — Claude API 호출 스펙

### 7-1. 요약 프롬프트 구조

```
System:
당신은 AI/ML 논문을 한국어로 요약하는 전문가입니다.
주어진 영문 초록을 정확하고 간결하게 3줄로 요약해주세요.

User:
다음 논문 초록을 한국어로 3줄 요약해주세요.

제목: {title}
초록: {abstract}

요약 형식:
1. [핵심 문제/목적]
2. [제안 방법/접근]
3. [주요 결과/기여]
```

### 7-2. summarizer.py 모듈 책임

| 책임 | 구현 방식 |
|------|-----------|
| Claude API 호출 | `anthropic.Anthropic().messages.create()` |
| 1차 실패 감지 | `anthropic.APIError`, `anthropic.APITimeoutError` 등 포착 |
| 재시도 (1회) | `time.sleep(retry_delay)` 후 동일 호출 |
| 최종 실패 폴백 | 원본 초록 반환 + `is_fallback: True` 플래그 |
| 로깅 | `logger.warning` (재시도) / `logger.error` (최종 실패) |

## 8. 중복 방지 구현 전략

```
파일 스캔 방식 (DB 없음):

1. output/*.md 파일 전체 읽기
2. 정규식으로 arXiv ID 파싱
   패턴: r"arXiv ID\*\*: (\d{4}\.\d{4,5})"
3. Set 자료구조로 기수집 ID 보관
4. 신규 논문 피드에서 ID 비교 → 중복 제외
```

> **수영 주석**: 파일이 수백 개 쌓이면 스캔 비용이 선형 증가하는 구조. 하루 10개 기준으로 1년치가 3,650개 논문 ID — 이 규모에서는 문제없음. "월 1,000건 이상" 요구사항이 생기면 SQLite로 교체 검토.

## 10. 실행 방식

### 수동 실행

```bash
python agent.py
python agent.py --date 2025-01-14   # 특정 날짜 지정 재실행
python agent.py --retry-only        # retry_queue.json의 실패 논문만 재시도
```

### cron 등록 예시 (매일 오전 9시)

```bash
0 9 * * * cd /path/to/arxiv-agent && python agent.py >> logs/agent.log 2>&1
```

### GitHub Actions (선택)

```yaml
on:
  schedule:
    - cron: '0 0 * * *'   # UTC 00:00 = KST 09:00
```

## 11. 확장 고려사항 (지금은 구현하지 않음)

| 기능 | 트리거 조건 | 전환 비용 |
|------|-------------|-----------|
| 중복 방지 DB화 | 일 수집 1,000건 초과 | 파일 스캔 → SQLite, arXiv ID 컬럼 추가 |
| 대시보드 HTML | stats JSON 구조 유지 중 | stats/YYYY-MM-DD.json을 그대로 읽어 렌더링 |
| Slack/Email 알림 | 요약 실패율 20% 초과 알림 필요 시 | agent.py 마지막에 알림 훅 추가 |
| 다중 요약 모델 | Claude 외 GPT-4o 비교 필요 시 | summarizer.py를 인터페이스 기반으로 교체 |
| 재시도 횟수 설정화 | 현재 하드코딩 1회 → 설정으로 | config.yaml `max_retries` 값 이미 준비됨 |
