## 핵심 설계 원칙

```
feedparser → Claude API (재시도 1회) → MD 저장
실패해도 원본 초록은 반드시 저장됨. 로그에 남김.
```

**팀 합의 사항 요약:**
- config 파싱 실패 = 앱 구동 실패 (전원 동의)
- Claude API 실패 = 재시도 1회 + 원본 초록 저장 (승우 제외 전원, 코드 6줄 차이라 수용)
- 중복 체크 = 파일 스캔 (복잡도 추가 없음)
- 대시보드, JSON 저장, 재수집 메커니즘 = **v2 이후** (민아님 양해 부탁)

### summarizer.py 핵심 로직

```python
import time
import logging

logger = logging.getLogger(__name__)

def summarize(title: str, abstract: str, client) -> str:
    """
    실패 시 1회 재시도. 그것도 실패 시 원본 초록 반환.
    """
    for attempt in range(2):
        try:
            response = client.messages.create(
                model="claude-opus-4-5",
                max_tokens=300,
                messages=[{
                    "role": "user",
                    "content": f"다음 논문 초록을 한국어로 3줄 요약해주세요.\n\n제목: {title}\n초록: {abstract}"
                }]
            )
            return response.content[0].text

        except Exception as e:
            if attempt == 0:
                logger.warning(f"[RETRY] '{title}' 요약 실패, 재시도 중: {e}")
                time.sleep(3)
            else:
                logger.error(f"[FAILED] '{title}' 요약 최종 실패, 원본 저장: {e}")

    return f"[요약 실패 - 원본 초록]\n{abstract}"
```

승우님이 우려한 "코드 복잡도": try-except 2번 + logger 2줄. 이게 전부입니다.

### agent.py 핵심 구조

```python
import sys
import yaml
import feedparser
import anthropic
from datetime import date
from pathlib import Path
from summarizer import summarize

# 1. config 로드 — 실패 시 즉시 종료
try:
    with open("config.yaml") as f:
        config = yaml.safe_load(f)
except Exception as e:
    print(f"[ERROR] config.yaml 파싱 실패: {e}")
    sys.exit(1)

keywords    = config["keywords"]
max_papers  = config.get("max_papers", 10)

# 2. arXiv 수집
papers = []
for keyword in keywords:
    feed = feedparser.parse(
        f"https://export.arxiv.org/api/query"
        f"?search_query=all:{keyword}&max_results={max_papers}"
        f"&sortBy=submittedDate&sortOrder=descending"
    )
    papers.extend(feed.entries)
    if len(papers) >= max_papers:
        break

papers = papers[:max_papers]

# 3. 중복 체크 — 파일 스캔
today     = date.today().isoformat()
out_path  = Path(f"output/{today}.md")

# 4. 요약 + MD 저장
client = anthropic.Anthropic()  # ANTHROPIC_API_KEY 환경변수 사용
lines  = [f"# arXiv 논문 수집 — {today}\n"]

for paper in papers:
    title    = paper.title
    abstract = paper.summary
    authors  = ", ".join(a.name for a in paper.authors[:3])
    link     = paper.link
    summary  = summarize(title, abstract, client)

    lines.append(f"## {title}")
    lines.append(f"**저자**: {authors}")
    lines.append(f"**링크**: {link}\n")
    lines.append(f"### 요약\n{summary}\n")
    lines.append("---\n")

out_path.parent.mkdir(exist_ok=True)
out_path.write_text("\n".join(lines), encoding="utf-8")
print(f"[DONE] {out_path} 저장 완료 ({len(papers)}개)")
```

## v2 범위 (운영 데이터 확인 후 결정)

> **판단 기준**: 1개월 운영 후 로그에서 "실제 문제"가 확인된 경우에만 진행.

| 항목 | 트리거 조건 |
|------|------------|
| HTML 대시보드 | "MD 파일 읽기 불편하다" 피드백 3회 이상 |
| 실패 논문 자동 재수집 | 요약 실패율 10% 초과 확인 시 |
| JSON 병렬 저장 | 다른 시스템 연동 요구가 생겼을 때 |
| 테스트 코드 | 리팩토링 시작 전 |
| config `error_handling` 섹션 | 재시도 정책 변경 요구 생겼을 때 |

수영님이 요청한 `error_handling` config 섹션: **지금은 정책이 "재시도 1회 + 원본 저장"으로 고정이라 설정 불필요**. 정책 변경 요구가 생기면 그때 추가합니다.
